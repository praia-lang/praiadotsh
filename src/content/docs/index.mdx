## Why use Praia?

**Broad stdlib.** HTTP client & server, JSON, YAML, SQLite, crypto, TCP/UDP, regex, and async are all built into the language. No package installs needed to start building.

**Pipe operator.** The `|>` operator turns nested calls into readable chains. Implicit `it` parameter removes boilerplate: `data |> filter(lam{ it > 5 }) |> map(lam{ it * 2 })`. The `|?>` error pipeline catches failures inline without try/catch nesting.

**Tagged values.** Data-carrying variants without declarations: `Ok(42)`, `Err("fail")`, `Point(1, 2)`. Pattern match to destructure: `match (result) { Ok(val) { ... } Err(msg) { ... } }`. A rare feature in dynamic languages.

**Safe regex.** When built with Google's RE2 engine, every regex completes in O(n) guaranteed time. No catastrophic backtracking. Safe for user-supplied patterns in web servers.

**Lightweight generators.** Generators use stack snapshots — no threads, no OS resources. Create thousands of them. They integrate directly with `for-in` loops for lazy evaluation.

**Precise numbers.** 64-bit integers stay exact. `10 / 2` returns `5` (int), not `5.0`. Overflow promotes to float instead of wrapping silently.

**Sane defaults.** Only `nil` and `false` are falsy. `UPPER_CASE` names warn on reassignment. `for-in` snapshots the collection so mutation mid-loop is safe.

**Pattern matching.** `match` supports equality, type patterns with `is`, guard clauses with `when`, and tagged value destructuring.

**Native C++ plugins.** Load shared libraries at runtime with `loadNative()`. Write performance-critical code in C++, expose it to Praia as regular functions. Scaffold a plugin project with `sand init --plugin`.

**Package manager.** [sand](https://github.com/praia-lang/sand) installs and manages grains (modules) from Git repositories.

```praia
// REST API with pipes
use "router"

let app = router.create()

app.get("/users/:id", lam{ req, params in
    let user = db.query(
        "SELECT * FROM users WHERE id = ?",
        [params.id]
    )
    return http.json(user[0])
})

app.get("/search", lam{ req in
    let results = sys.read("data.json")
        |> json.parse
        |?> lam{ [] }
        |> filter(lam{ it.active })
        |> sort(lam{ a, b in a.name < b.name })
    return http.json(results)
})

app.listen(8080)
```

```praia
// Tagged values + pattern matching
func divide(a, b) {
    ensure (b != 0) else { return Err("division by zero") }
    return Ok(a / b)
}

let results = [10, 0, 5]
    |> map(lam{ divide(100, it) })

for (r in results) {
    match (r) {
        Ok(val) { print("result: " + str(val)) }
        Err(msg) { print("error: " + msg) }
    }
}
```

```praia
// WebSocket chat server
use "socket"

let clients = []

let server = http.createServer(lam{ req in
    if (req.path == "/ws") {
        return socket.upgrade(req, lam{ ws in
            clients.push(ws)
            ws.onMessage(lam{ msg in
                clients
                    |> filter(lam{ it.id != ws.id })
                    |> each(lam{ it.send(msg) })
            })
            ws.onClose(lam{ in
                clients = clients
                    |> filter(lam{ it.id != ws.id })
            })
        })
    }
    return {body: "Chat server"}
})
server.listen(8080)
```

```praia
// CLI tool — log analysis
#!/usr/bin/env praia
use "colors"

let logs = sys.readLines("access.log")
    |> map(lam{ it.split(" ") })
    |> filter(lam{ len(it) >= 3 })
    |> map(lam{ {ip: it[0], status: it[1], path: it[2]} })

let byStatus = logs |> groupBy(lam{ it.status })

for ({key, value} in byStatus) {
    let color = key.startsWith("2") ? colors.green
              : key.startsWith("4") ? colors.yellow
              : colors.red
    print("%{color(key)}: %{len(value)} requests")
}
```
