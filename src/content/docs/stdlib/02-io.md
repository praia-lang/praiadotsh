Praia splits OS-level operations across two namespaces:

- **`fs`** -- filesystem I/O (read/write/exists/mkdir/...)
- **`sys`** -- process concerns (env, cwd, exec, signals, terminal)

The split is recent. The filesystem ops used to live on `sys` (`sys.read`, `sys.write`, etc.) and the old names still work, but they print a one-shot deprecation warning on first use per process. Rename `sys.<op>` to `fs.<op>` whenever you touch a file; the aliases will be removed at 1.0.

## File I/O

```praia
// Write a file
fs.write("output.txt", "hello from praia")

// Read a file
let content = fs.read("output.txt")
print(content)          // hello from praia

// Append to a file
fs.append("output.txt", "\nmore text")

// Read as array of lines (no trailing newlines)
for (line in fs.readLines("output.txt")) {
    print(line)
}
```

## File system

```praia
fs.mkdir("my/nested/dir")          // creates all parent dirs
print(fs.exists("output.txt"))     // true
fs.remove("output.txt")            // delete a file
fs.remove("my/nested/dir")         // delete a directory (recursive)
fs.copy("src.txt", "dst.txt")      // copy a file
fs.copy("srcdir", "dstdir")        // copy a directory (recursive)
fs.move("old.txt", "new.txt")      // move / rename
let files = fs.readDir("my/dir")   // array of filenames in directory
```

## Metadata, permissions, links

```praia
let s = fs.stat("config.toml")
print(s.type)   // "file" | "dir" | "symlink" | "socket" | "fifo" | "block" | "char" | "unknown"
print(s.size)   // bytes
print(s.mode)   // low 12 bits: permission + setuid/setgid/sticky
print(s.mtime)  // unix seconds; also atime, ctime
print(s.uid, s.gid, s.nlink, s.ino, s.dev)

// lstat doesn't follow symlinks — use it to detect them.
if (fs.exists("backup") && fs.lstat("backup").type == "symlink") {
    print("backup is a symlink to", fs.readlink("backup"))
}

// Permission bits via chmod. Praia has no octal literal yet so write
// the decimal (or hex) equivalent.
fs.chmod("secret.key", 384)   // 0o600
fs.chmod("script.sh", 493)    // 0o755

// Symlinks. Target string is stored verbatim — dangling is legal.
fs.symlink("/usr/local/bin/praia", "/tmp/p")
print(fs.readlink("/tmp/p"))  // "/usr/local/bin/praia"
```

## Atomic writes and temp files

```praia
// Crash-safe write: temp file in the same directory, fsync, then
// rename(2). Readers see either the old contents or the new ones —
// never a half-written file.
fs.atomicWrite("config.toml", encoded)

// Race-free temp directory (mkdtemp, mode 0700).
let workDir = fs.tempDir("myapp")
try {
    // ... do work in workDir
} finally {
    fs.remove(workDir)
}

// Race-free temp FILE (mkstemp, mode 0600).
let tmp = fs.mktemp("upload")
fs.write(tmp, body)
// ... use tmp ...
fs.remove(tmp)
```

Use `fs.atomicWrite` for any file where a partial write would corrupt downstream readers — config, lockfiles, cache. The temp file goes in the target's directory so the rename is genuinely atomic (POSIX `rename(2)` on the same filesystem).

The temp helpers default to a `praia` prefix when called without arguments.

## File handles (streaming I/O)

`fs.open(path, mode)` returns a file handle for streaming reads and writes. Reach for it when the file is too large to slurp into memory, when you need random access via `seek`, or when you want to interleave reads and writes on the same fd.

```praia
// Line-by-line over an arbitrarily large file
let h = fs.open("big.log", "r")
defer h.close()
let line = h.readLine()
while (line != nil) {
    if (line.contains("ERROR")) { print(line) }
    line = h.readLine()
}

// Random access with seek + tell
let h = fs.open("data.bin", "r+")
h.seek(-4, "end")             // last 4 bytes
let footer = h.read(4)
h.seek(0)                     // back to start
h.write("PATCHED")
h.close()
```

| Mode | Behavior |
|------|----------|
| `"r"`  | Read only; file must exist |
| `"w"`  | Write only; truncates or creates (mode 0644) |
| `"a"`  | Write only, append; creates if missing |
| `"r+"` | Read + write; file must exist |
| `"w+"` | Read + write; truncates or creates |
| `"a+"` | Read + write; writes always append |

### Handle methods

| Method | Description |
|--------|-------------|
| `h.read(n)` | Read up to `n` bytes; `""` at EOF |
| `h.readLine()` | Read one line (no trailing `\n`); `nil` at EOF |
| `h.write(data)` | Write string/bytes; returns bytes written |
| `h.seek(offset, whence?)` | Reposition. `whence` is `"start"` (default), `"current"`, or `"end"` |
| `h.tell()` | Current logical position (accounts for the read-ahead buffer) |
| `h.flush()` | `fsync(2)` — push the page cache to disk |
| `h.close()` | Close the fd. Idempotent. Subsequent reads/writes throw |
| `h.path` | Path the handle was opened on |
| `h.mode` | Mode string the handle was opened with |

`read` and `readLine` share an internal 4 KiB buffer so you can mix them on the same handle without losing bytes. Writes on `r+`/`w+` handles automatically resync the kernel position after an intervening read, so the write lands at the logical position you expect.

## Running commands

`sys.exec(cmd)` (also `sys.run(cmd)`) runs a command and returns `{stdout, stderr, exitCode}`. Pass a string for shell execution, or an array for safe argument passing:

```praia
// String form -- runs through /bin/sh
let r = sys.exec("ls -la")
print(r.stdout)
print(r.exitCode)          // 0

// Array form -- no shell, safe from injection
let r2 = sys.exec(["git", "clone", "--quiet", url, dest])
print(r2.exitCode)

// Check for errors
let r3 = sys.exec("cat nonexistent.txt")
if (r3.exitCode != 0) {
    print("Error:", r3.stderr)
}
```

## Process spawning

`sys.spawn(cmd)` launches a child process with piped stdin/stdout/stderr. Pass a string for shell execution, or an array for safe argument passing (no shell, no injection):

```praia
// String form — runs via /bin/sh -c (shell expansion applies)
let proc = sys.spawn("cat -n")
proc.write("hello\n")
proc.write("world\n")
proc.closeStdin()
print(proc.read())
print("exit:", proc.wait())

// Array form — runs via execvp (each element is an exact argv entry)
let proc2 = sys.spawn(["ffmpeg", "-i", userPath, "-f", "mp3", "out.mp3"])
```

### Process handle methods

| Method | Description |
|--------|-------------|
| `proc.write(data)` | Write to the child's stdin |
| `proc.closeStdin()` | Close stdin (signals EOF) |
| `proc.read()` | Read all of stdout (blocks until EOF) |
| `proc.readErr()` | Read all of stderr |
| `proc.readLine()` | Read one line from stdout (nil on EOF) |
| `proc.wait()` | Wait for exit, returns exit code |
| `proc.kill(signal?)` | Send a signal (default SIGTERM) |
| `proc.pid` | The child's process ID |

### Line-by-line reading

```praia
let proc = sys.spawn("grep -i error")
proc.write("INFO all good\n")
proc.write("ERROR disk full\n")
proc.write("ERROR timeout\n")
proc.closeStdin()

let line = proc.readLine()
while (line != nil) {
    print("match:", line)
    line = proc.readLine()
}
proc.wait()
```

Use `sys.exec` for fire-and-forget commands. Use `sys.spawn` when you need to pipe input, read output line-by-line, or interact with a long-running process.

## Command-line arguments

Arguments after the script name are available in `sys.args`:

```sh
praia script.praia hello world
```

```praia
print(sys.args)         // ["hello", "world"]
```

## Reading from stdin

`sys.input(prompt?)` reads one line from standard input. Returns `nil` on EOF (Ctrl-D).

```praia
let name = sys.input("What's your name? ")
if (name == nil) { sys.exit(0) }
```

## Environment variables

```praia
print(sys.env("HOME"))             // "/Users/ada"
sys.setenv("MY_VAR", "hello")
print(sys.cwd())                   // "/path/to/project"
print(sys.platform)                // "darwin"
```

| Function | Description |
|----------|-------------|
| `sys.env(name)` | Read environment variable (nil if not set) |
| `sys.envAll()` | Returns all environment variables as a map |
| `sys.setenv(name, value)` | Set an environment variable |
| `sys.cwd()` | Current working directory |
| `sys.chdir(path)` | Change working directory |
| `sys.uid()` | Effective user ID |
| `sys.isRoot()` | `true` if running as root (uid 0) |
| `sys.getpid()` | Current process ID |
| `sys.platform` | `"darwin"`, `"linux"`, or `"windows"` |

## Exiting

```praia
sys.exit(0)             // exit with code 0
sys.exit(1)             // exit with code 1
```

### CLI exit code convention

When `praia` itself decides the exit code (no explicit `sys.exit`), it uses:

| Code | Meaning |
|------|---------|
| `0` | Success |
| `1` | Usage error — bad CLI args, file not found, missing `-c` argument |
| `2` | Compile error — lex / parse / bytecode codegen failure |
| `3` | Runtime error — uncaught exception, VM or interpreter failed |
| `N` | `sys.exit(N)` from your program — passes through unchanged |

`praia test` is the exception: it exits `0` if all tests passed, `1` if any failed (CI-friendly).

## Signal handling

Register callbacks for OS signals:

| Function | Description |
|----------|-------------|
| `sys.onSignal(name, fn)` | Register a handler. Pass `nil` to restore default. |
| `sys.checkSignals()` | Process pending signals. Returns `true` if any fired. |
| `sys.signal(name)` | Send a signal to the current process. |

Supported: `SIGINT`, `SIGTERM`, `SIGHUP`, `SIGUSR1`, `SIGUSR2` (short names like `INT` also work).

```praia
let running = true
sys.onSignal("SIGINT", lam{ sig in
    print("shutting down...")
    running = false
})

while (running) {
    // ... do work ...
    sys.checkSignals()
}
print("cleanup done")
```

Call `sys.checkSignals()` in long-running loops so callbacks get a chance to run.

## Terminal I/O

```praia
sys.rawMode(true)           // disable line buffering
let key = sys.readKey()     // read a single keypress
sys.rawMode(false)          // restore normal terminal

let size = sys.termSize()
print(size.rows, size.cols)
```

| Function | Description |
|----------|-------------|
| `sys.rawMode(bool)` | Enable/disable raw terminal mode |
| `sys.readKey()` | Read a single keypress (handles escape sequences) |
| `sys.termSize()` | Returns `{rows, cols}` |
| `sys.stdout(str)` | Write to stdout without trailing newline |

Arrow keys return escape sequences: `"\x1b[A"` (up), `"\x1b[B"` (down), `"\x1b[C"` (right), `"\x1b[D"` (left).

## Path utilities

| Function | Description |
|----------|-------------|
| `path.join(parts...)` | Join path segments |
| `path.dirname(p)` | Parent directory |
| `path.basename(p)` | Filename component |
| `path.ext(p)` | File extension (including dot) |
| `path.resolve(p)` | Absolute path |
| `path.isFile(p)` | `true` if path is a regular file |
| `path.isDir(p)` | `true` if path is a directory |
| `path.size(p)` | File size in bytes |
| `path.mtime(p)` | Last modification time (seconds since the Unix epoch) |
| `path.glob(dir, pattern)` | Match files by pattern (e.g. `"*.praia"`) |

```praia
path.join("src", "main.cpp")       // "src/main.cpp"
path.dirname("/usr/local/bin/p")   // "/usr/local/bin"
path.basename("/usr/local/bin/p")  // "p"
path.ext("script.praia")          // ".praia"
```
